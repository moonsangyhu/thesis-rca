# Paper Survey — AIOps 논문 조사

최근 3년(2023-2026) AIOps + LLM 관련 논문에서 성능 개선이 보고된 기법을 조사하고, 실험 계획에 활용할 수 있는 인사이트를 정리한다.

## 인자

- 실험 버전 번호 (예: `v5`) — 결과 파일 네이밍에 사용
- 선택적: 특정 주제 (예: "context engineering", "prompt optimization")

## Workflow

### 1. 검색 키워드로 WebSearch 수행

아래 키워드를 **모두** 검색한다:

```
"LLM root cause analysis kubernetes" 2023 2024 2025 2026
"AIOps large language model cloud" 2023 2024 2025 2026
"automated incident diagnosis microservices LLM" 2023 2024 2025
"context engineering LLM operations" 2024 2025
"prompt optimization root cause analysis" 2024 2025
"retrieval augmented generation AIOps" 2024 2025
```

인자로 특정 주제가 주어졌으면 해당 주제 관련 키워드도 추가한다.

### 2. 논문 선별

검색 결과에서 **최소 5편 이상** 선별한다. 선별 기준:
- 클라우드/K8s 환경에서 LLM을 활용한 장애 진단(RCA) 연구
- 정량적 성능 개선이 보고된 논문 (정확도 수치 포함)
- 2023년 이후 발표

### 3. 논문별 정보 수집

각 논문에 대해 아래 정보를 수집한다:

| 항목 | 설명 |
|------|------|
| 제목 | 논문 제목 (영문) |
| 저자 | 주요 저자 (1-3명) |
| 연도/학회 | 출판 연도, 학회/저널명 |
| 핵심 기법 | 사용한 주요 기법 (예: CoT, RAG, multi-agent, context reranking 등) |
| 대상 시스템 | 실험 대상 (K8s, 마이크로서비스, 클라우드 등) |
| 사용 모델 | 사용한 LLM 모델 |
| 보고된 효과 | 정확도 개선 수치 (예: +15pp, 65%→80%) |
| 본 실험 적용 가능성 | 우리 실험에 적용할 수 있는 구체적 인사이트 |
| URL | 논문 링크 (arXiv, ACM, IEEE 등) |

### 5. 완료 보고

- 조사된 논문 수와 핵심 인사이트를 요약 보고
- `/changelog` 호출
- `/commit-push` 호출

## Rules

- **최소 5편 이상** 논문을 조사해야 한다
- WebSearch 결과가 부족하면 키워드를 변형하여 재검색한다
- 논문의 정량적 결과(정확도 수치)가 없으면 선별에서 제외한다
- 기존 조사 결과(`docs/surveys/paper_survey_v*.md`)가 있으면 읽고, 이전에 조사한 논문은 중복 포함하지 않는다
- 결과 문서는 한국어로 작성하되, 논문 제목과 기법명은 영문 유지
